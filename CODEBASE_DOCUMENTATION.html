1. ğŸ¯ Project Overview

connectNALCO is a comprehensive digital workplace portal designed specifically for NALCO (National Aluminium Company Limited) operations. It serves as a unified platform for employee management, issue tracking, attendance monitoring, and operational excellence.

ğŸ¯ Primary Objectives:

  • Digital Transformation: Modernize NALCO's traditional paper-based processes
  • Employee Empowerment: Provide self-service capabilities for common HR tasks
  • Operational Efficiency: Streamline workflows and reduce manual intervention
  • Data-Driven Insights: Enable analytics and reporting for better decision making
  • Role-Based Access: Secure, hierarchical access control system
Target Users: Employees (2,500+), Department Authorities (50+), System Administrators (10+)

8. ğŸ” Authentication System

JWT-Based Authentication

The application implements a secure JWT (JSON Web Token) based authentication system with role-based access control.

ğŸ”‘ Authentication Flow

  1. Login Request: User submits credentials via login form
  2. Credential Validation: Server verifies email and password against database
  3. Token Generation: JWT token created with user ID and role
  4. Token Storage: Client stores token in localStorage with expiration
  5. Request Authorization: Token included in API request headers
  6. Token Validation: Server validates token and extracts user context
  7. Session Management: Automatic logout on token expiration

ğŸ‘¥ Role-Based Access Control

Role Access Level Permissions Default Routes
Employee Basic View own data, submit requests, track attendance /portal/*
Authority Departmental Manage team, approve requests, view reports /authority/*
Admin System-wide Full system access, user management, configurations /admin/*

ğŸ›¡ï¸ Security Features

  • Password Hashing: bcryptjs with 12 salt rounds
  • Token Expiration: 8-hour session timeout
  • Route Protection: ProtectedRoute component for access control
  • CORS Configuration: Controlled cross-origin access
  • Input Validation: Zod schema validation
  • SQL Injection Prevention: MongoDB parameterized queries
  • XSS Protection: React built-in sanitization
